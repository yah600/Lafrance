### **Lesson 10.4: Handling Common Situations**

**SITUATION 1: Emergency Call During Busy Time**
```
Problem: Client has burst pipe, all techs busy

Solution:
1. Check GPS map
2. Find tech closest to completion
3. Chat tech: "How long until done?"
4. Assign emergency as next job
5. Call client with ETA
6. Mark job "URGENT" in Kanban
```

---

**SITUATION 2: Technician Calls In Sick**
```
Problem: Jean is sick, has 4 jobs today

Solution:
1. Go to Jean's profile
2. View today's jobs
3. Reassign jobs:
   - Use auto-dispatch for automatic
   - Or manually drag to other techs
4. Chat each affected tech
5. Call clients to update ETAs
6. Set Jean's status to "Hors service"
```

---

**SITUATION 3: Client Disputes Invoice**
```
Problem: Client says charge too high

Solution:
1. Go to client detail
2. Review job history
3. Check service performed
4. Review invoice line items
5. If error:
   - Create credit note
   - Regenerate correct invoice
6. If no error:
   - Explain charges
   - Offer payment plan if needed
7. Escalate to manager if unresolved
```

---

**SITUATION 4: Multiple Urgent Jobs at Once**
```
Problem: 3 urgent calls in 10 minutes

Solution:
1. Create all jobs quickly
2. Mark all as "URGENT"
3. Check GPS for available techs
4. Assign based on location:
   - Closest tech to each job
5. Stagger ETAs
6. Consider calling in off-duty tech
7. Keep clients updated
```
